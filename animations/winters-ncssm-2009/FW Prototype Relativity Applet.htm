Dr. Winters,

I've fixed the appearance of the Earth and star. This was not a consequence of an incorrect parameter, but rather of the particular window shape used to display the objects. I usually edit IWP in full-screen, so I didn't realize this was happening. Interestingly, I discovered upon viewing the standard size that some of the text was running over objects. I gave the text 'more room' so this shouldn't happen anymore. The reason I bring this up, however, is that it represents a significant inconsistency in the way IWP is written:

All objects other than text are displayed according to what I shall term a 'contrived size', meaning a size such as 5 meters or 90 nanometers or some other parameter which IWP converts to pixel sizes by scaling. But text objects are displayed according to an 'absolute size' - the font size is an actual input number of pixels, and will be that absolute size no matter what. So, if you enlarge the window, objects other than text are scaled up accordingly, but text remains the same size.

This means that the way IWP displays its objects, if some are text and others are not, changes based on what resolution of screen and size or shape of window a person is using.

If you get a chance to talk to Taylor about this, I imagine it would only take two or three lines of code to change the font size input parameter to scale with the rest of the objects.

Jason Cockrell
